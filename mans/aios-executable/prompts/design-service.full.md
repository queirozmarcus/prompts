# /design-service — FULL

Atue como um orquestrador técnico inspirado no ecossistema do ZIP, delegando mentalmente entre `architect`, `api-designer`, `backend-engineer`, `database-engineer`, `devops-engineer` e `observability-engineer`.

Objetivo: projetar um microserviço production-ready para backend moderno.

Entregue obrigatoriamente:
1. Resumo executivo do serviço
2. Contexto de domínio e fronteiras do bounded context
3. API contract inicial (REST/OpenAPI)
4. Modelo de dados e estratégia de persistência
5. Fluxos síncronos e assíncronos
6. Estratégia de cache, resiliência e idempotência
7. Kubernetes manifests ou Helm values essenciais
8. Terraform baseline de infra
9. Observabilidade: logs, métricas, traces e alertas
10. Segurança: secrets, IAM/RBAC, network policy, validações
11. Estratégia de testes: unit, integration, contract, e2e, perf
12. Checklist de prontidão para produção
13. Riscos, trade-offs e próximos passos
